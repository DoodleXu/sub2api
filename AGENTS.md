# 开发约定

1. 用于保持最佳的开发和生产环境的最佳实践。
2. 任何时候，只要你被要求审核代码，都要对审核结果给出针对性的、最合理的建议。
3. 保持良好的项目结构，持续保持模块化编程和高可维护性。
4. 每次 commit 都要按 `xxx: xxxx` 的格式，用中文总结 commit 内容，例如：`chore: 准备发布 v0.1.148`。

## Fork 差异台账

- 本 fork 相对上游官方仓库的自定义功能差异维护在 `docs/FORK_UPSTREAM_DIFF_CN.md`。
- 后续同步上游时，必须先参考该台账判断需要保留的 fork 行为；同步完成后，如基线、风险点、验证结果或 fork 行为发生变化，必须同步更新台账。
- 后续在本 fork 中新增、修改或删除自定义功能时，也必须维护该台账，尤其是签到、运营中心、成本核算、账号归档、Web 创作台、生图管理等 fork 定制能力。
